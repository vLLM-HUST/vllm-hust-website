Practice

Development path

Students work from reproducible observations to small patches, benchmark records, and reviewable pull requests.

Step 1

Find a concrete issue

Start from CI failures, benchmark gaps, workload metadata, runtime behavior, documentation, or small scheduling and KV-cache problems.

Step 2

Reproduce and measure

Record commit, model, hardware, driver, startup parameters, workload, token sizes, concurrency, and dtype before making claims.

Step 3

Submit a focused PR

Keep patches narrow, explain the affected path, include tests or benchmark evidence, and document any remaining risk.
